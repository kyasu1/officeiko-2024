# GEMINI.md

## Project Overview

This is an **Elm Pages v3** application, designed to be deployed on **Netlify**. It uses **Elm** for the frontend, **Tailwind CSS v4** for styling, and relies on **Netlify Functions** for server-side capabilities. The project appears to be a website (likely for "Office Iko" based on the directory name and content), possibly utilizing a headless CMS or local Markdown content.

### Key Technologies

*   **Framework:** `elm-pages` (v3)
*   **Language:** Elm (v0.19.1)
*   **Styling:** Tailwind CSS (v4) via Vite plugin
*   **Build Tool:** Vite (v7)
*   **Deployment:** Netlify (using `elm-pages/adapter/netlify.js`)
*   **Backend/Content:** 
    *   Netlify Functions (`functions/`)
    *   Markdown content (`content/`)
    *   Potentially Strapi (referenced in `README.md`)

## Building and Running

### Prerequisites
*   Node.js (version managed by `.nvmrc` if present, otherwise LTS recommended)
*   Elm (installed via `elm-tooling`)

### Commands

*   **Install Dependencies:**
    ```bash
    npm install
    ```
    *Note: This automatically runs `elm-tooling install`.*

*   **Start Development Server:**
    ```bash
    npm start
    ```
    *Note: This command expects a `.env` file to be present (`cat .env | xargs`). If it fails, ensure `.env` exists.*

*   **Build for Production:**
    ```bash
    npm run build
    ```
    *This runs `elm-pages build`, generating the `dist/` directory for deployment.*

## Directory Structure

*   `app/`: Main application source code.
    *   `Route/`: File-based routing modules (e.g., `Index.elm`, `About.elm`).
    *   `View.elm`, `Site.elm`: Core `elm-pages` configuration modules.
*   `src/`: Shared Elm modules and business logic (e.g., `Market.elm`, `Ticket.elm`).
*   `content/`: Content files (Markdown/JSON) used by the application.
*   `functions/`: Netlify Functions (Serverless functions).
*   `codegen/`: Code generated by `elm-codegen`.
*   `.elm-pages/`: Internal `elm-pages` generated files.
*   `public/`: Static assets (images, favicons).
